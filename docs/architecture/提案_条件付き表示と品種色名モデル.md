# 設計メモ: 条件付き表示と品種色名モデル

> **位置づけ**: 本メモは実装前の設計提案（DRAFT）。合意後に V9 正本群へ反映する。
> 直前に完了した「タビーA展開（カテゴリA化）＋ Egyptian Mau/Ocicat の A/A 固定」を土台とする。

---

## 1. 背景・目的

通常モードの予測結果を、**遺伝学的に正しく・分かりやすく・学びになる**形にする。特に:

- 「隠れキャリア次第で出る色」を、平坦な 50:50 の単一数字で誤魔化さず、**条件付き（＝どの保因があれば出るか）**として見せる。
- その色が出た事実から**両親の隠れ遺伝子型を逆推論**できる、という繁殖実務に役立つ情報を提供する。
- 品種固有の色名（例: アビシニアンの Ruddy/Blue…）を正しく表示する。

---

## 2. 決定済みの方針（ロック）

| # | 決定 |
|---|------|
| D1 | **カテゴリA座位（A タビー・D 希釈・I シルバー・Mc・Ta）の X/- 展開計算は行う**（変化を失わないため）。ただしその結果は**メインではなく「もし出たら」枠に逆推論で表示**する（§3）。展開自体を止めると出力が親の表現型に縮むため計算は維持。 |
| D2 | **カテゴリC座位（B シナモン/チョコ・C ポイント/セピア）は原則キャリア非展開**（表現型に固定）。 |
| D3 | ただし **片親が表現型で劣性を発現しているときだけ**、相方の保因を条件提示する（表現型トリガー型）。例: Ruddy×Sorrel → 相方Ruddyを B/bl として受け入れ、シナモン系を条件付きで出す。 |
| D4 | 品種として柄が固定される猫種（Egyptian Mau/Ocicat=Sp/Sp、Abyssinian等=A/A・Ta/Ta）は、品種制約で親遺伝子型を絞りソリッド等を出さない。 |
| D5 | 出力は「**確定して出る色**」と「**もしこの色が出たら**」の2セクションに分ける。 |
| D6 | 「もしこの色が出たら」の色は**色系統でまとめ**、**逆推論ラベル**（この色が出た＝両親が◯◯を保因）を付ける。 |

---

## 3. 出力モデル定義（確定）

### 3.1 メイン結果（確定して出る）

- **隠れキャリアを仮定しない「確定色」のみ**。親の表現型から必ず出る色。
- 例: Ruddy × Ruddy → Ruddy。Blue × Blue → Blue。
- 目的: 「まず確実にこうなる」を明快に示す。

### 3.2 「もしこの色が出たら」セクション（畳み・逆推論）

- **対象**: カテゴリA由来のキャリア依存色（Blue=D/d、ソリッド=a/a、シルバー等）＋ **セピア（C/cb）** ＋ **シナモン/チョコ（B/bl・b/b、差分に出たとき）**。
- **除外**: **ポイント（cs）** は色数が多く溢れるため出さない。シナモン/チョコが**存在しない品種には出さない**。
- **表示**: 色系統でまとめ、各グループに**逆推論ラベル**「この色が出たら両親◯◯確定 → X%」。畳みで普段は邪魔しない。
- **実装土台**: 既存 `carrier_exploration` の「シナリオ＋new_colors＋ラベル」を流用。どの親が何を保因すれば出るかは `_carrier_options_for` が算出済み。

```
Ruddy × Ruddy
─────────────
確定して出る
  Ruddy

▸ もしこの色が出たら（畳み）
  Blue        → 両親 D/d 確定なら 25%
  セピア系     → 両親 C/cb 確定
  シナモン系   → 両親 B/bl 確定
  （※ポイントは多すぎるので出さない）
```

### 3.3 committed 済み実装との関係

- 直前コミット（`19edf66`）で A をカテゴリA展開し、通常メインにソリッドが出る状態にした。
- 本モデルでは、その**カテゴリA由来の色（ソリッド/Blue等）は「メイン」から「もし出たら」へ表示位置を移す**。
- 展開計算そのものは流用するため、エンジン実装は無駄にならない（＝表示層の再構成）。

## 4. 品種色名システム

### 4.1 現状の問題（命名バグ）

アビシニアン Ruddy×Ruddy の希釈個体が **「Blue Tabby」** と generic 表示される。Ta/Ta（ティックド）が命名に反映されず、品種色名にも変換されていない。

### 4.2 あるべき姿（アビシニアン／ソマリ）

固有値 **Ta/Ta＝必ずティックドタビー**。表示名は柄名ではなく品種の色名を使う:

| 品種色名 | 遺伝的中身 |
|---------|-----------|
| Ruddy | 黒ベース・濃色（B_ D_） |
| Red（Sorrel） | シナモン・濃色（bl/bl D_） |
| Blue | 黒ベース・希釈（B_ d/d） |
| Fawn | シナモン・希釈（bl/bl d/d） |
| AOC | 上記以外（シルバー等）＝ Any Other Color |

= **B座位（黒 vs シナモン）× D座位（濃 vs 希釈）** の4象限 ＋ AOC。

### 4.3 やること

- 品種文脈で、命名結果を上記の品種色名へ正しく変換（`display_alias_map` 経由）。
- 「Blue Tabby」等の generic 名が品種指定時に出ないようにする。

## 5. 文脈注釈（特別な状況の説明）

- パターンに応じて、結果に**特別な状況の説明**を出す（例:「この品種は常にティックドタビー。表示名は Ruddy/Blue…」「A/A固定でソリッドは出ない」）。
- 既存 `ParentColorNotes` / `NormalModeNote` の仕組みを拡張。
- **気づいた所から段階的に**追加する方針（一括ではなく継続的に）。

## 7. 決定事項（確認済み）

1. **範囲**: カテゴリA色（Blue/ソリッド等）は**メインに出さず「もし出たら」に逆推論でまとめる**。メインは確定色のみ。
2. **セピア**: 「もし出たら」に**入れる**（品種・色数が少なく散らからない）。両親 C/cb 確定。
3. **ポイント**: **入れない**（色数が多すぎて溢れる）。
4. **シナモン/チョコ**: 差分に出たら出す。ただし**その色が存在しない品種には出さない**。
5. **逆推論ラベル**: シンプルに「この色が出たら両親◯◯保因確定（X/x）→ X%」。
6. **品種色名の着手**: アビシニアン → ソマリの順。他品種は段階展開。
7. **AOC**: 一括扱い（Silver 等を個別展開しない）。

### 残る細部（実装時に詰める）
- 逆推論ラベルの最終的な日本語文言・レイアウト。
- 「もし出たら」の色系統グルーピングの粒度（何をひとまとめにするか）。
