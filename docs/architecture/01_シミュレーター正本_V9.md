# 猫の遺伝学シミュレーター 正本 V9
作成日: 2026-06-24  
位置づけ: **計算ロジック・出力仕様・検証基準の唯一正本**

---

## 0. 本書の目的

本書は、猫の毛色・柄の交配シミュレーターにおける、計算ロジック、入力前提、キャリア扱い、表現型出力、検証基準を定義する唯一正本である。

旧V8では「表現型だけから全潜在キャリアを自動展開する」思想が含まれていたが、これは通常計算の結果に未確定キャリア由来の不適切な毛色名を混入させる危険がある。V9では以下を明確に分離する。

- 通常モード
- 明示キャリアモード

> **モード改訂 (V9.x)**: かつて存在した「全キャリア探索モード (carrier_exploration)」は廃止した。
> 通常モードの結果レポートが **周辺確率 `results` (カテゴリA 展開込み) + 推定色 `conditional_color_groups`
> (「もしこの色が出たら親が◯◯保因と確定」)** を常時提示するようになり、探索モードの目的
> (隠れキャリア由来カラーの可視化) を包含・上回るため。詳細は §2.3。

本シミュレーターは、**CSVに存在する名前を返すこと**を目的としない。  
目的は、**入力条件から論理的に導出可能な毛色・柄候補と確率を、遺伝学的矛盾なく返すこと**である。

## 1. スコープ

### 1.1 対象

フェーズ1で扱う対象は、猫のカラー・パターン計算に直接関係する以下の要素である。

- B Locus: ブラック系列
- D Locus: 希釈
- A Locus: アグチ
- O Locus: オレンジ、伴性遺伝
- C Locus: フルカラー、ポイント、セピア、ミンク
- W Locus: 優性白
- S Locus: 白斑
- Mc Locus: マッカレル / クラシック
- Ta Locus: ティックド
- Sp Locus: スポット
- I Locus: シルバー / スモーク
- Wb Locus: ワイドバンド系

### 1.2 対象外

フェーズ1では以下を通常計算に含めない。

- XXYオス
- キメラ
- 体細胞モザイク
- 突然変異による例外
- 繁殖医学、疾患、骨格、耳、足長など毛色計算外の形態変異
- 実際の血統集団におけるキャリア頻度推定
- 表現型だけでは確定できない隠れキャリアの確率的推定

## 2. 計算モード定義

### 2.1 normal: 通常モード

通常UIおよび通常APIで使用する標準モード。

#### 入力

- 父毛色
- 母毛色
- 猫種、任意
- 明示キャリアなし

#### ルール

通常モードでは、以下のみを計算に使う。

1. 親の表現型から確定できる遺伝子条件 (優性形質は **§2.4 優性表現型のヘテロ未確定ルール** に従い X/- として扱う)
2. 猫種から固定される遺伝子条件
3. 性別から確定するO遺伝子条件

以下の「表現型から直接要求されない潜在キャリア」は勝手に展開してはならない。

- B/b チョコレートキャリア
- B/bl シナモンキャリア
- C/cs ポイントキャリア
- C/cb セピアキャリア

> 注 (V9.2 改訂): 濃色 (D)・シルバー (I)・**タビー (A)** など **優性形質が表現されている座** は、
> 表現型だけからホモ接合と断定できないため、通常モードでも X/- = {X/X, X/x} の両方を
> 計算対象に含める (§2.4 カテゴリA)。例えば Silver Tabby の父は I/- なので非シルバーの子
> (Brown 系) を、D/- なので希釈の子 (Blue 系) を正しく生み得る。
> **A (タビー) も同様にカテゴリA として X/- = {A/A, A/a} を展開する** (仕様変更: 旧 V9.2 は A を
> カテゴリA' として A/A 相当に固定していたが、これを廃止)。**理由**: タビー猫は表現型からは A/A (純ホモ)
> か A/a (ソリッドを隠し持つヘテロ) かを区別できない。両親がともに A/a の場合、子の一部が a/a (ソリッド)
> になるのは遺伝学的に正しい (両親が A/a である確率 0.25 × その場合の子の a/a 率 0.25 = 約 6.25%)。
> よって通常モードでもソリッドの子を正しい確率で出す。**逆方向の不可逆ルールは維持される**: 非オレンジの
> a/a × a/a は配偶子に A を持たないため、そこからタビー A/- は生まれない (§4.2)。A 展開により低確率の
> ソリッド / スモーク / トーティ系が増えるが、これは遺伝的に正しい出力であり、表示側 (UI) で低確率カラーを
> アコーディオン集約して抑制する。B/C 系の潜在キャリアは引き続き明示キャリアモードでのみ扱う (通常モードでも推定色 `conditional_color_groups` として「もし出たら」提示はされる)。

入力表現型から白斑・シルバー・希釈などが明示されている場合は、その表現型の成立に必要な範囲で候補を持たせる。

例:

- Blue は d/d 確定 (劣性固定)
- Silver Tabby は I/- かつ D/- かつ A/- (いずれも優性ヘテロ未確定)。A は normal_mode でも X/- = {A/A, A/a} を展開する
- -White は S/s 確定。Van (S/S) は白斑レベルが上位の別表現型であり、明示されない限り出さない
- Patched / Pt / Tortie はメス O/o 確定。O/o のトーティは normal_mode では原則 Patched Tabby (タビートーティ) として出るが、両親が A/a のとき a/a の子は Calico / Tortoiseshell (ソリッドトーティ) として低確率で出る (A 展開に伴う遺伝的に正しい出力)

### 2.2 explicit_carrier: 明示キャリアモード

ユーザーが親の隠れキャリアを明示した場合にのみ使用する。

例:

```json
{
  "sire_color": "Silver Tabby",
  "dam_color": "Blue Pt Tabby-White",
  "breed": "Any",
  "mode": "explicit_carrier",
  "sire_carriers": {
    "C": "C/cs",
    "B": "B/b"
  },
  "dam_carriers": {
    "C": "C/cs"
  }
}
```

この場合のみ、ポイント・チョコレート等の未表現キャリア由来カラーを通常結果に含めてよい。

### 2.3 (廃止) carrier_exploration: 全キャリア探索モード

**V9.x で廃止**。かつては「理論上隠し持ち得るキャリアを総当たりする研究・検証用モード」として、
結果を通常結果 `results` と分離し `carrier_exploration_results` に返していた。

**廃止理由**: 通常モードの結果レポートが以下を常時提示するようになり、探索モードの目的
(隠れキャリア由来カラーの可視化) を包含・上回るため。

1. **周辺確率 `results`** — カテゴリA (A/D/I/Mc/Ta) を X/- 展開した周辺分布 (§2.4)。
2. **推定色 `conditional_color_groups`** — 確定色 (`confirmed_results`) に無いが隠れキャリアがあれば
   出る色を、逆推論ラベル (「この色が出たら親が◯◯保因と確定」) 付きで色系統ごとに提示。

`conditional_color_groups` のシナリオは、旧 carrier_exploration の「片親劣性発現 × 相手キャリア」に加え、
「両親とも優性発現 → 両親ヘテロ仮定」も含み、カバー範囲がより広い。かつ推定色の計算経路は
carrier_exploration の経路から独立しているため、後者を撤去しても推定色レポートは影響を受けない。

> **移行**: モードは `normal` / `explicit_carrier` の2つ。`carrier_exploration` を指定する API 呼び出しは
> 受け付けない (422)。`carrier_exploration_results` レスポンスフィールドは廃止。

## 2.4 優性表現型のヘテロ未確定ルール (Dominant Expressed Unknown Rule)

### 規則

> 優性形質が表現されている遺伝子座のうち A・D・I などは、表現型のみから優性ホモ接合であるとは
> 断定しない。たとえばタビーは A/A ではなく A/-、濃色は D/D ではなく D/-、シルバーは I/I ではなく I/- として扱う。
>
> 通常モードでは、これらの「表現型を成立させる優性アレルのヘテロ可能性」は計算対象に含める。
> A (タビー) も D/I/Mc/Ta と同じカテゴリA として X/- = {A/A, A/a} を展開する
> (旧 V9.2 のカテゴリA' 固定は廃止。理由は下記 §A.タビーの A-locus を参照)。
> 一方でチョコレートキャリア B/b、シナモンキャリア B/bl、ポイントキャリア C/cs、セピアキャリア C/cb など、
> 表現型から直接必要とされない潜在キャリアは、明示キャリア情報または血統・産子履歴がない限り通常結果には含めない。

一言でまとめると:

**A・D・I など表現型に現れている優性形質は 優性/- として扱いホモ接合に固定しない (A も X/- = {A/A, A/a} を展開する)。表現型から直接要求されない潜在キャリア (B/b, B/bl, C/cs, C/cb) は通常モードでは勝手に展開しない。**

### A. 表現型確定・優性ヘテロ不確定座位 (通常モードでも展開する)

優性形質が見えているが、ホモかヘテロか表現型からは分からない座。X/- = {X/X, X/x} を両方計算対象にする。

| 座位 | 表現型 | 扱い |
|---|---|---|
| A | タビーなら A/- | A/A または A/a（下記 A. タビーの A-locus）|
| D | 濃色なら D/- | D/D または D/d |
| I | シルバーなら I/- | I/I または I/i |
| Mc | マッカレルなら Mc/- | Mc/Mc または Mc/mc |
| Ta | ティックドなら Ta/- | Ta/Ta または Ta/ta |
| W | **入力が White のときのみ** W/- | W/W または W/w（下記 A''）|

実装: `master_data.py` `_build_normal_parent_genotypes` の `dominant_expandable` に A/D/I/Mc/Ta/W を持たせる。
`val == (優性, 優性)` のときだけヘテロ候補を足すため、CSV で W/W を持つのは White のみ = 他色は不変。
`NORMAL_OPENED_LOCI = ("A","D","I","Mc","Ta")`・`NORMAL_CLOSED_LOCI = ("B","C","Wb")` を順方向診断メタデータに
用い、White の順方向レポートだけ `opened_loci` に "W" を加える (下記 A'')。

### A''. White (優性白) の W-locus (入力が White のときのみ展開)

優性白 (White) は下の色を完全にマスクするため、表現型からは W/W か W/w かも、下の色 (B/D/A/C/O/I/S/Wb)
も一切分からない。

- **展開の入口を White 入力に絞る**: W/- 展開を無条件に行うと白斑 S や他座位の解決に副作用が出るため、
  CSV で W/W を持つ White のみに作用させる (他色の W は w/w で不変)。
- **順方向 (normal) は W/w を仮定した専用集計 + AOC 集約** (§4.5 / §6.1)。単純な 50:50 展開だと
  白 75% になるが、有色の可能性を見せるため White 親を W/w と仮定し、白 50%・有色 50% を出す。
  有色のうち母由来で確定できる部分 (父 White のときの有色オス＝母の色) だけを色名で出し、
  White(下不明)側から受け継ぐ有色 (父 White の有色メス、母 White の有色オス・メス) は色が
  定まらないため **AOC (Any Other Color)** に集約する。AOC は実在の毛色ではないため canonical
  正規化・display_alias_map の対象に含めない (§6.1)。
- **逆引き・リター推定は W/w 展開で逆算する**。White 親から色付きの子が出た実績は「親が w を渡せる
  ＝ W/w」を確定する (§4.5 の不可逆ルールの逆命題)。リター推定では White 親の下地 (下不明) を
  座位別に逆算する (`litter_inference._white_locus_analysis`)。全下地の遺伝子型 (オス約7万/メス約10万)
  を列挙せず、観察子猫が拘束する座位だけを座位ごとに突き合わせ、拘束されない下地は「下不明」と
  明示する。これにより「White 親は a/a ソリッド」等の誤確定を防ぎ、計算量・メモリも抑える。

### A'→A. タビーの A-locus (旧カテゴリA' を廃止しカテゴリA として展開する)

タビー表現型 (A/-) は normal_mode でも **X/- = {A/A, A/a}** を展開する (D/I/Mc/Ta と同じカテゴリA)。

| 座位 | 表現型 | normal_mode の扱い |
|---|---|---|
| A | タビーなら A/- | A/A または A/a (X/- 展開) |

理由 (仕様変更): タビー猫は表現型からは A/A (純ホモ) か A/a (ソリッドを隠し持つヘテロ) かを区別できない。
両親がともに A/a の場合、子の約 25% が a/a (ソリッド) になるのは遺伝学的に正しく、これを通常モードでも
正しい確率で出すべきである (血統情報が無い通常モードでは各親の A/A:A/a = 50:50 の中立仮定。両親が
A/a である確率 0.25 × その場合の子の a/a 率 0.25 = 約 6.25% がソリッド)。旧 V9.2 では a/a×a/a 由来の
Solid / Tortie / Calico / Tortoiseshell / Smoke 誤出力を避けるため A を A/A 相当に固定していた (カテゴリA')
が、これは正しい確率のソリッドの子も潰してしまうため廃止する。

**逆方向の不可逆ルールは維持される** (§4.2): 非オレンジの a/a × a/a は配偶子に A を持たないため、そこから
タビー A/- は生まれない。A の X/- 展開はあくまで「タビー親がソリッドを隠し持ち得る」順方向の話であり、
不可逆ルール (ソリッド×ソリッド→タビー禁止) とは両立する。

**暴発対策**: A 展開で低確率のソリッド / スモーク / トーティ系が増えるが、これは遺伝的に正しい出力である。
表示側 (UI) で低確率カラーをアコーディオン集約して抑制する (UI 詳細は別途)。なお A/a を明示したい場合は
explicit_carrier_mode を引き続き利用できる (通常モードでも A/a 由来の色は推定色レポートで「もし出たら」提示される)。
(O/o のメスは、A/- のときタビー文脈で Patched Tabby として出る。両親 A/a 由来の a/a の子のみ
Calico / Tortoiseshell として低確率で出る。)

### B. 表現型確定・劣性固定座位 (固定する)

| 表現型 | 遺伝子型 |
|---|---|
| Blue / Cream / Lilac / Fawn | d/d |
| Solid | a/a |
| Siamese Point | cs/cs |
| Burmese Sepia | cb/cb |
| Tonkinese Mink | cb/cs |

### C. 表現型からは未確認の潜在キャリア (通常モードでは展開しない)

| 潜在キャリア | 通常モード |
|---|---|
| B/b チョコキャリア | 明示・履歴がなければ出さない |
| B/bl シナモンキャリア | 明示・履歴がなければ出さない |
| C/cs ポイントキャリア | 明示・履歴がなければ出さない |
| C/cb セピアキャリア | 明示・履歴がなければ出さない |

### ヘテロ展開の確率重み付け

カテゴリA の X/- 展開は、血統情報が無い通常モードでは **X/X : X/x = 50 : 50 の中立仮定** で重み付けする
(例: 濃色の親が D/d である確率を 50% とする)。実集団のキャリア頻度はこれより低いことが多いが、
通常モードでは中立仮定を採用する。将来的に、親の明示キャリア指定 (explicit_carrier) を UI から
入力できるようにし、その場合はユーザー指定値で上書きする。

### S (白斑) の扱い — 特記

S 白斑は **不完全優性** であり、S/S (Van / 高白斑) と S/s (バイカラー / -White) は表現型で区別できる。
したがって白斑の接合性は入力の白斑レベルから確定し、A/D/I のような「ヘテロ不可視」座とは性質が異なる。
本実装では S を X/- として展開せず、入力名 (-White=S/s, Van=S/S, 無印=s/s) のまま固定する。
これにより、S/s の子が逆引きで Van 名へ誤マッチする問題も回避する。

> 確定 (ユーザー承認済み): 白斑は入力レベルで確定する。"White" が入っていれば S/s (-White)、
> "Van" が入っていれば S/S (Van)、無印は s/s。S は X/- 展開しない。

### 130 × 204 への帰結

父 Silver Tabby = `O:o/Y, A:A/-, I:I/-, D:D/-, S:s/s, C:C/C, B:B/B`、母 Blue Pt Tabby-White =
`O:O/o, D:d/d, A:A/-, I:i/i, S:S/s, C:C/C, B:B/B` の場合、通常モードで次が正しく出る
(A も X/- 展開するため、多数派のタビー / パッチドタビー系に加え、両親 A/a 由来の a/a ソリッド系が低確率で出る)。

- 父が I/i なら非シルバー → Brown Tabby / Brown Tabby-White (父 I/i + 母 O/o のメスは Brown Patched Tabby)
- 父が D/d なら希釈 → Blue Tabby / Blue Silver Tabby (父 D/d + 母 O/o のメスは Blue (Silver) Patched Tabby)
- 母が O/o なのでメスにパッチド → Silver Patched Tabby / Brown Patched Tabby / Blue (Silver) Patched Tabby 系

A を X/- 展開するため、両親がともに A/a のときに限り a/a の子が生じ、Solid / Smoke (a/a) や、O/o メスでは
Tortie / Calico / Tortoiseshell (a/a 前提のソリッドトーティ) が **低確率で** 出る (両親とも Silver/Blue Tabby=A/- で
血統情報が無いため、両親 A/a 確率 0.25 × 子 a/a 率 0.25 = 約 6.25% がソリッド系)。これは遺伝的に正しい出力であり、
表示側 (UI) で低確率カラーをアコーディオン集約して抑制する。一方、B/C を展開しないため Chocolate / Cinnamon /
Lilac / Fawn / Point / Mink / Sepia / Lynx Point は引き続き出ない。逆方向 (§4.2 不可逆ルール) は維持され、
a/a×a/a からタビー A/- は生まれない。

|---|---|---|

## 4. 遺伝子座と不可逆ルール

### 4.1 D Locus

- D/-: 濃色
- d/d: 希釈

不可逆ルール:

- d/d × d/d から D/- は生まれない

### 4.2 A Locus

- A/-: タビー許可
- a/a: ソリッド

不可逆ルール:

- 非オレンジの a/a × a/a から A/- タビーは生まれない
- オレンジ系列ではAに関係なくタビー様表現が顕在化する

### 4.3 O Locus

- オス: O/Y または o/Y
- メス: O/O, O/o, o/o

不可逆ルール:

- o/Y × o/o から Red, Cream, Tortie, Calico は生まれない
- 通常XYオスに Tortie, Tortoiseshell, Calico, Blue Cream, Lilac Cream 等を出してはならない

### 4.4 C Locus

- C/-: フルカラー
- cs/cs: Point
- cb/cb: Sepia
- cb/cs: Mink

不可逆ルール:

- cs/cs × cs/cs から C/- は生まれない
- cb/cb × cb/cb から C/- は生まれない

normalでの制限:

- 親または猫種または明示キャリアにC制限がない場合、Point/Mink/Sepiaを勝手に出してはならない

### 4.5 W Locus

- W/-: White
- w/w: 有色

不可逆ルール:

- w/w × w/w から W/- は生まれない

normalでの扱い (§2.4 A''):

- W は「入力表現型が White のときのみ」W/w に展開する (他色は w/w 固定・不変)。
- 順方向 (normal) は White 親を W/w と仮定した専用集計を行い、下記の割合を出す
  (下の色が未確定な部分は **AOC (Any Other Color)** に集約)。

期待割合 (片親のみ White):

| ケース | 白オス | 白メス | 色付きオス | 色付きメス |
|---|---|---|---|---|
| 父 White × 母 既知色 | 25% | 25% | 母の色 25% | AOC 25% |
| 母 White × 父 既知色 | 25% | 25% | AOC 25% | AOC 25% |

- 父 White: オスは母由来で色が確定する (オスは母の X=O座位 と母の優性形質で色が決まり、父の下地は
  乗らない扱い)。メスは White(下不明)父の X を受け継ぐため、その下地アレル (O=赤の可能性、母が
  劣性ホモの座位で父の優性が乗るか等) が定まらず色を確定できない → 有色メスは全て AOC。
  両親 White は白 75% + AOC 25%。
- リター推定: White 親から色付きの子が出たら W/w を確定 (矛盾にしない)。下の色は子から逆算する
  範囲で確定/条件付き/未確認に分類し、White 親の下地は全不明として扱う (§2.4 A'')。

### 4.6 S Locus

- S/-: 白斑あり
- s/s: 白斑なし

不可逆ルール:

- s/s × s/s から白斑ありは生まれない

normalでの出力:

- 一般白斑は `-White`
- `Van`, `Mitted`, `Bi-Color` は明示または猫種文脈がない限り出さない

### 4.7 Mc Locus

- Mc/-: Mackerel
- mc/mc: Classic

不可逆ルール:

- mc/mc × mc/mc から Mc/- は生まれない

normalでの出力:

- 親入力に Mackerel / Classic が明示されない場合、通常表示では単に `Tabby` へ簡略化してよい

### 4.8 Ta Locus

- Ta/-: Ticked
- ta/ta: 通常タビー

不可逆ルール:

- ta/ta × ta/ta から Ta/- は生まれない

normalでの出力:

- 猫種が Abyssinian / Somali 等でない限り、Tickedを勝手に出さない

### 4.9 Sp Locus

- Sp/-: Spotted
- sp/sp: 非スポット

不可逆ルール:

- sp/sp × sp/sp から Sp/- は生まれない

### 4.10 I Locus

- I/-: Silver / Smoke
- i/i: 非シルバー

不可逆ルール:

- i/i × i/i から Silver / Smoke は生まれない

### 4.11 Wb Locus

- Wb/-: Wide Band
- wb/wb: 通常幅

不可逆ルール:

- wb/wb × wb/wb から Chinchilla / Shaded / Shell / Golden 系を出してはならない

## 6. 出力名ルール

### 6.1 基本方針

出力名は以下の順に決定する。

1. 内部遺伝子型から標準表現型を算出
2. normal / explicit_carrier のモードを確認
3. 性別制約を適用
4. 猫種がある場合は猫種別表示名へ変換
5. 猫種がない場合は一般表示名に正規化
6. 出力禁止語・文脈不一致語を除外
7. 表示確率を集計

**AOC (Any Other Color) の特例 (§2.4 A'' / §4.5)**: White 親の下の色が未確定な有色カテゴリは
`AOC` として通常の色行と同じ形式で1行表示する。AOC は実在の毛色ではないため上記1〜7の命名
パイプライン (canonical 正規化・display_alias_map・猫種別表示名) を通さず、順方向集計の最後段に
そのまま付与する。UI は AOC 行にフォーカス（ホバー/タップ）したときだけ説明ツールチップを開き、デフォルトでは
説明を出さない (下の色を入力すれば explicit_carrier で確定する、という導線を添える)。

### 6.2 CSVに存在する名前の扱い

CSVに存在するカラー名であっても、以下に該当する場合はnormal結果に出してはならない。

- 親の表現型から導出できない
- 明示キャリアがない
- 猫種文脈がない
- 性別と矛盾する
- 白斑レベルが明示されていない
- 通常表示では猫種固有名に該当する

### 6.3 normalで禁止する代表例

- Mink
- Sepia
- Point、明示C条件なしの場合
- Mitted、猫種文脈なしの場合
- Bi-Color、猫種文脈なしの場合
- Van、親または白斑レベル明示なしの場合
- Champagne
- Platinum
- Sable
- Choco省略表記
- male Tortie
- male Tortoiseshell
- male Calico
- male Blue Cream
- male Lilac Cream

### 6.4 ワイドバンド (tipping) 系の命名

ワイドバンド (`Wb/-`) は **非オレンジ・アグーチ背景でのみ tipping (Shell / Shaded / Chinchilla / Golden) として発現**する。`normal_mode` で Wb キャリアを自動展開はしない (§4.11 不可逆ルール) が、親がワイドバンドで子が実際に `Wb/-` になる場合は、その子を**未分類にせず命名する**。

- **非オレンジ・アグーチ・`Wb/-`**: tipping 色として命名する。
  - シルバー (`I/-`) → `Silver`、非シルバー (`i/i`) → `Golden`。希釈 (`d/d`) は `Blue` を冠する。
  - tipping の**濃淡** (Shell < Shaded < Chinchilla) は多遺伝子 (rufousing/wideband polygene) で 1 座位に還元できないため、**親カラー名から推論**する (タビー柄 Mackerel/Classic を親名から決めるのと同方針)。親が `Chinchilla` / `Shaded` / `Shell` を含めばそれを冠し、無ければ濃淡なし (`Golden` / `Silver`)。
  - 例: `Chinchilla Golden × Black` → 非オレンジの子に `Chinchilla Golden` / `Blue Chinchilla Golden` が出る。
- **オレンジ / トーティ / ソリッドの `Wb/-`**: tipping は固有名を持たないため通常命名へ流す (ソリッドでは tipping 非発現、赤系は `Cameo` / `Cream` 等で命名)。
  - 例: `Chinchilla Golden × Cream Tabby-White` は赤ダムのため子が全て赤/トーティになり、純 `Golden` は遺伝的に出ない (`Cameo` / `Patched Tabby` 等で命名され、未分類にはならない)。

> tipping は本来ポリジーンで連続的なため、ここでの命名は「Wb/- の子を消さずに代表名へ寄せる」実務的規約である。母数となる遺伝モデルの厳密化は将来拡張。

## 7. 診断情報

分類不能な子猫遺伝子型を無言で破棄してはならない。

内部診断として以下を保持する。

```json
{
  "diagnostics": {
    "mode": "normal",
    "matched_probability": 100.0,
    "unmatched_probability": 0.0,
    "unmatched_genotype_count": 0,
    "unmatched_samples": []
  }
}
```

### 7.1 再正規化ルール

`unmatched_probability > 0` の状態で、残った結果だけを100%に再正規化してはならない。

通常結果として100%表を返してよいのは、以下の場合のみ。

- `unmatched_probability == 0`
- または未分類を明示的に `Unclassified` として返す

## 8. 確率表示ルール

### 8.1 0.0%禁止

丸め後に `0.0%` となる項目は通常UIに表示しない。

API内部で保持する場合は、以下のいずれかとする。

- `raw_probability`
- `probability_pct`
- `display_probability`

通常表示では `min_display_probability_pct` 未満を除外する。

### 8.2 合計確率

表示結果の合計は100%になること。

ただし、未分類結果を捨てて再正規化した100%は不合格とする。

### 9.3 目標カラーから探す API (逆引きMVP)

`POST /api/v1/reverse-lookup` は、登録猫または自舎カラー構成と目標カラーから、目標カラーが
成立し得る交配候補を返す。既存 `POST /api/v1/calculate` の互換性を壊さないため、新規
エンドポイントとして追加する。

リクエストは `target_color` を必須とし、`target_sex` (`male` / `female`) は任意とする。
`target_sex` が指定された場合、確率は該当性別の子猫結果だけを目標条件として集計する。
登録猫の `color` は通常シミュレーターと同じ `cat_color_master.csv` の alias / canonical 解決を通し、
親の性別と `SexRestriction` が矛盾する場合は候補計算へ進まず 422 とする。保存済みデータやAPI直送でも、
父猫にメス限定カラーを指定するなどの不整合を黙って候補なしへ落とさない。

逆引きでは、通常シミュレーターの `normal` 結果とは別に、以下を分けて表示する。

- `confirmed_probability_pct`: 登録済み毛色・確認済み因子だけで評価できる確定確率
- `conditional_max_probability_pct`: 未確認因子が成立すると仮定した条件付き最大確率
- `confirmation_needed`: 確認が必要な条件
- `recommended_tests`: 推奨検査
- `locus_evidence`: 座位別根拠
- `other_possible_colors`: 目標カラー以外に生まれる可能性のあるカラー

逆引き結果は、UIで以下のカテゴリに分けて表示する。

1. 確定で期待できる
2. 条件付きで期待できる
3. 現在の情報では判定が難しい
4. 現在の登録情報では確認できない

候補が見つからない場合も、単に候補なしとはせず、目標カラーに必要な主な条件、現在確認できない
条件、確認するとよい項目を返す。ゴールデン / ワイドバンド / CORIN 系は品種・系統差が大きいため、
現在の対応範囲では既存ルールに基づく参考情報として扱い、断定しすぎない。

**入力上限・計算量 (実装メモ)**: 登録猫は最大50頭まで (`field_validator` で超過は422、暴走防止)。
内部はオス×メスの総当たりだが、`CoatColorCalculator` の純粋計算 (親遺伝子型解決・配偶子生成・
`calculate_report`) をプロセス内メモ化し、同一色組の再計算を排除する (出力は不変)。

### 9.4 リター実績から推定 API (MVP)

`POST /api/v1/litter-inference` は、父猫・母猫の表現型と実際に生まれた子猫群のカラー・性別から、
観察実績を説明できる父母遺伝子型候補を絞り込む。既存 `POST /api/v1/calculate` と
`POST /api/v1/reverse-lookup` の互換性を壊さないため、新規エンドポイントとして追加する。

親猫と子猫の `color` は通常シミュレーターと同じ `cat_color_master.csv` の alias / canonical 解決を
通す。親候補生成・Punnett 分離・子猫遺伝子型の構築は `CoatColorCalculator` の共通処理を使い、
色名・遺伝子座の読み込み先を別系統にしない。

レスポンスは、観察された全子猫を説明できる親候補ペアを基準に、座位別結果を以下へ分類する。

- `confirmed`: 全候補で共通するため確定できる因子
- `conditional`: 遺伝子型は割れるが、観察実績を説明するうえで共通する条件
- `inferred`: 観察実績から強く支持される推定
- `unconfirmed`: 候補が割れて追加確認が必要な因子
- `contradictions`: どの候補でも説明できない矛盾
- `warnings`: Red / Cream 系のA座位判定、Calico / Tortie 系の白斑判定などの注意

MVPでは、タビー細分 (Mc / Ta / Sp) は既存命名ロジック同様に親名・表示後処理の影響が大きいため、
観察実績の主照合から外す。Calico / Tortie 系で白斑呼称が混在する場合は、S座位を断定せず警告と推奨確認を返す。

**入力上限・計算量 (実装メモ)**: 観察子猫は最大12頭まで (`field_validator` で超過は422、暴走防止)。
親候補ペアの照合は、メンデル分離が座位ごとに独立であることを用いて座位単位のビットマスク照合へ
分解する (結合した子猫遺伝子型集合を物質化しない)。これにより最悪ケース (例: Silver Tabby ×
Silver Tabby) でも実用速度 (実機で約0.3秒) で応答する。判定結果 (生存ペア集合・findings) は
従来の総当たり実装と同一で、新旧等価テストで担保する。
