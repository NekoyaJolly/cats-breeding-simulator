# 猫の遺伝学シミュレーター 正本 V9
作成日: 2026-06-24  
位置づけ: **計算ロジック・出力仕様・検証基準の唯一正本**

---

## 0. 本書の目的

本書は、猫の毛色・柄の交配シミュレーターにおける、計算ロジック、入力前提、キャリア扱い、表現型出力、検証基準を定義する唯一正本である。

旧V8では「表現型だけから全潜在キャリアを自動展開する」思想が含まれていたが、これは通常計算の結果に未確定キャリア由来の不適切な毛色名を混入させる危険がある。V9では以下を明確に分離する。

- 通常モード
- 明示キャリアモード
- 全キャリア探索モード

本シミュレーターは、**CSVに存在する名前を返すこと**を目的としない。  
目的は、**入力条件から論理的に導出可能な毛色・柄候補と確率を、遺伝学的矛盾なく返すこと**である。

## 1. スコープ

### 1.1 対象

フェーズ1で扱う対象は、猫のカラー・パターン計算に直接関係する以下の要素である。

- B Locus: ブラック系列
- D Locus: 希釈
- A Locus: アグチ
- O Locus: オレンジ、伴性遺伝
- C Locus: フルカラー、ポイント、セピア、ミンク
- W Locus: 優性白
- S Locus: 白斑
- Mc Locus: マッカレル / クラシック
- Ta Locus: ティックド
- Sp Locus: スポット
- I Locus: シルバー / スモーク
- Wb Locus: ワイドバンド系

### 1.2 対象外

フェーズ1では以下を通常計算に含めない。

- XXYオス
- キメラ
- 体細胞モザイク
- 突然変異による例外
- 繁殖医学、疾患、骨格、耳、足長など毛色計算外の形態変異
- 実際の血統集団におけるキャリア頻度推定
- 表現型だけでは確定できない隠れキャリアの確率的推定

## 2. 計算モード定義

### 2.1 normal: 通常モード

通常UIおよび通常APIで使用する標準モード。

#### 入力

- 父毛色
- 母毛色
- 猫種、任意
- 明示キャリアなし

#### ルール

通常モードでは、以下のみを計算に使う。

1. 親の表現型から確定できる遺伝子条件 (優性形質は **§2.4 優性表現型のヘテロ未確定ルール** に従い X/- として扱う)
2. 猫種から固定される遺伝子条件
3. 性別から確定するO遺伝子条件

以下の「表現型から直接要求されない潜在キャリア」は勝手に展開してはならない。

- B/b チョコレートキャリア
- B/bl シナモンキャリア
- C/cs ポイントキャリア
- C/cb セピアキャリア

> 注 (V9.2 改訂): 濃色 (D)・シルバー (I) など **優性形質が表現されている座の一部** は、
> 表現型だけからホモ接合と断定できないため、通常モードでも X/- = {X/X, X/x} の両方を
> 計算対象に含める (§2.4 カテゴリA)。例えば Silver Tabby の父は I/- なので非シルバーの子
> (Brown 系) を、D/- なので希釈の子 (Blue 系) を正しく生み得る。
> **ただし A (タビー) は例外で、normal_mode では A/A 相当に固定し A/a を展開しない** (§2.4 カテゴリA')。
> A/a を展開すると a/a×a/a が成立し、タビー親から Solid / Tortie / Calico / Smoke (a/a 前提) が
> 出てしまうため。A/a と B/C 系の潜在キャリアは明示キャリア / 全キャリア探索モードでのみ扱う。

入力表現型から白斑・シルバー・希釈などが明示されている場合は、その表現型の成立に必要な範囲で候補を持たせる。

例:

- Blue は d/d 確定 (劣性固定)
- Silver Tabby は I/- かつ D/- (優性ヘテロ未確定)。A (タビー) は normal_mode では A/A 相当に固定 (A/a 非展開)
- -White は S/s 確定。Van (S/S) は白斑レベルが上位の別表現型であり、明示されない限り出さない
- Patched / Pt / Tortie はメス O/o 確定。ただし O/o のトーティは normal_mode では Patched Tabby (タビートーティ) として出る。a/a 前提の Calico / Tortoiseshell (ソリッドトーティ) は A を展開しないため出さない

### 2.2 explicit_carrier: 明示キャリアモード

ユーザーが親の隠れキャリアを明示した場合にのみ使用する。

例:

```json
{
  "sire_color": "Silver Tabby",
  "dam_color": "Blue Pt Tabby-White",
  "breed": "Any",
  "mode": "explicit_carrier",
  "sire_carriers": {
    "C": "C/cs",
    "B": "B/b"
  },
  "dam_carriers": {
    "C": "C/cs"
  }
}
```

この場合のみ、ポイント・チョコレート等の未表現キャリア由来カラーを通常結果に含めてよい。

### 2.3 carrier_exploration: 全キャリア探索モード

理論上隠し持ち得るキャリアを総当たりする研究・検証用モード。

このモードの結果は、通常結果 `results` とは分離し、`carrier_exploration_results` として返す。

通常UIではデフォルト非表示とする。

#### 禁止事項

carrier_exploration の結果を normal の結果として表示してはならない。

> **実装状況 (Phase 2, 実装済)**: carrier_exploration は **「片親が劣性形質を完全発現している
> 場合に、相手がそのキャリアだったらどうなるか」のみ**を条件付きシナリオとして提示する。
> 対象座位は A(a/a)・B(b/b, bl/bl)・C(cs/cs, cb/cb, cb/cs)・D(d/d)。**両親とも通常表現型で
> 両方が隠れキャリアかもしれない、という探索は自動生成しない** (組み合わせ爆発と誤出力の再発防止)。
> I / Wb / tipping は初期対象外。`carrier_probability` 入力は実装しない。**事前確率 (キャリア頻度)
> は一切掛けず**、各シナリオは条件付き (`probability_basis="conditional_on_other_parent_carrier"`,
> `prior_probability_applied=false`)。結果は `results` (normal) と分離して `carrier_exploration_results`
> 配列で返す (各要素: scenario / label / assumed_carriers / results / new_colors)。

## 2.4 優性表現型のヘテロ未確定ルール (Dominant Expressed Unknown Rule)

### 規則

> 優性形質が表現されている遺伝子座のうち D・I などは、表現型のみから優性ホモ接合であるとは
> 断定しない。たとえば濃色は D/D ではなく D/-、シルバーは I/I ではなく I/- として扱う。
>
> 通常モードでは、これらの「表現型を成立させる優性アレルのヘテロ可能性」は計算対象に含める。
> 一方で、A (タビー) は normal_mode では A/A 相当に固定し A/a を展開しない (カテゴリA')。
> またチョコレートキャリア B/b、シナモンキャリア B/bl、ポイントキャリア C/cs、セピアキャリア C/cb など、
> 表現型から直接必要とされない潜在キャリアも、明示キャリア情報または血統・産子履歴がない限り通常結果には含めない。

一言でまとめると:

**D・I など表現型に現れている優性形質は 優性/- として扱いホモ接合に固定しない。ただし A (タビー) は normal_mode では A/A 相当に固定する。表現型から直接要求されない潜在キャリア (A/a, B/b, B/bl, C/cs, C/cb) は通常モードでは勝手に展開しない。**

### A. 表現型確定・優性ヘテロ不確定座位 (通常モードでも展開する)

優性形質が見えているが、ホモかヘテロか表現型からは分からない座。X/- = {X/X, X/x} を両方計算対象にする。

| 座位 | 表現型 | 扱い |
|---|---|---|
| D | 濃色なら D/- | D/D または D/d |
| I | シルバーなら I/- | I/I または I/i |
| Mc | マッカレルなら Mc/- | Mc/Mc または Mc/mc |
| Ta | ティックドなら Ta/- | Ta/Ta または Ta/ta |

### A'. タビーの A-locus (通常モードでは展開しない)

タビー表現型 (A/-) は normal_mode では **A/A 相当**として扱い、A/a を展開しない。

| 座位 | 表現型 | normal_mode の扱い |
|---|---|---|
| A | タビーなら A/- | **A/A に固定** (A/a 非展開) |

理由: A/a を展開すると a/a×a/a が成立し、タビー親から a/a 前提の Solid / Tortie / Calico /
Tortoiseshell / Smoke が出てしまう。これは表現型から要求されない潜在キャリアの自動展開であり、
通常モードでは禁止する。A/a は explicit_carrier_mode または carrier_exploration_mode でのみ使う。
(O/o のメスは、タビー文脈では Patched Tabby として出る。Calico/Tortoiseshell ではない。)

### B. 表現型確定・劣性固定座位 (固定する)

| 表現型 | 遺伝子型 |
|---|---|
| Blue / Cream / Lilac / Fawn | d/d |
| Solid | a/a |
| Siamese Point | cs/cs |
| Burmese Sepia | cb/cb |
| Tonkinese Mink | cb/cs |
| 非シルバー | i/i |
| 白斑なし | s/s |

### C. 表現型からは未確認の潜在キャリア (通常モードでは展開しない)

| 潜在キャリア | 通常モード |
|---|---|
| B/b チョコキャリア | 明示・履歴がなければ出さない |
| B/bl シナモンキャリア | 明示・履歴がなければ出さない |
| C/cs ポイントキャリア | 明示・履歴がなければ出さない |
| C/cb セピアキャリア | 明示・履歴がなければ出さない |

### ヘテロ展開の確率重み付け

カテゴリA の X/- 展開は、血統情報が無い通常モードでは **X/X : X/x = 50 : 50 の中立仮定** で重み付けする
(例: 濃色の親が D/d である確率を 50% とする)。実集団のキャリア頻度はこれより低いことが多いが、
通常モードでは中立仮定を採用する。将来的に、親の明示キャリア指定 (explicit_carrier) を UI から
入力できるようにし、その場合はユーザー指定値で上書きする。

### S (白斑) の扱い — 特記

S 白斑は **不完全優性** であり、S/S (Van / 高白斑) と S/s (バイカラー / -White) は表現型で区別できる。
したがって白斑の接合性は入力の白斑レベルから確定し、A/D/I のような「ヘテロ不可視」座とは性質が異なる。
本実装では S を X/- として展開せず、入力名 (-White=S/s, Van=S/S, 無印=s/s) のまま固定する。
これにより、S/s の子が逆引きで Van 名へ誤マッチする問題も回避する。

> 確定 (ユーザー承認済み): 白斑は入力レベルで確定する。"White" が入っていれば S/s (-White)、
> "Van" が入っていれば S/S (Van)、無印は s/s。S は X/- 展開しない。

### 130 × 204 への帰結

父 Silver Tabby = `O:o/Y, A:A/A(固定), I:I/-, D:D/-, S:s/s, C:C/C, B:B/B`、母 Blue Pt Tabby-White =
`O:O/o, D:d/d, A:A/A(固定), I:i/i, S:S/s, C:C/C, B:B/B` の場合、通常モードで次が正しく出る
(A を展開しないため出力は全てタビー / パッチドタビー系)。

- 父が I/i なら非シルバー → Brown Tabby / Brown Tabby-White (父 I/i + 母 O/o のメスは Brown Patched Tabby)
- 父が D/d なら希釈 → Blue Tabby / Blue Silver Tabby (父 D/d + 母 O/o のメスは Blue (Silver) Patched Tabby)
- 母が O/o なのでメスにパッチド → Silver Patched Tabby / Brown Patched Tabby / Blue (Silver) Patched Tabby 系

一方、A を展開しないため a/a 前提の Solid / Tortie / Calico / Tortoiseshell / Smoke は出ない。
B/C を展開しないため Chocolate / Cinnamon / Lilac / Fawn / Point / Mink / Sepia / Lynx Point も出ない。

## 4. 遺伝子座と不可逆ルール

### 4.1 D Locus

- D/-: 濃色
- d/d: 希釈

不可逆ルール:

- d/d × d/d から D/- は生まれない

### 4.2 A Locus

- A/-: タビー許可
- a/a: ソリッド

不可逆ルール:

- 非オレンジの a/a × a/a から A/- タビーは生まれない
- オレンジ系列ではAに関係なくタビー様表現が顕在化する

### 4.3 O Locus

- オス: O/Y または o/Y
- メス: O/O, O/o, o/o

不可逆ルール:

- o/Y × o/o から Red, Cream, Tortie, Calico は生まれない
- 通常XYオスに Tortie, Tortoiseshell, Calico, Blue Cream, Lilac Cream 等を出してはならない

### 4.4 C Locus

- C/-: フルカラー
- cs/cs: Point
- cb/cb: Sepia
- cb/cs: Mink

不可逆ルール:

- cs/cs × cs/cs から C/- は生まれない
- cb/cb × cb/cb から C/- は生まれない

normalでの制限:

- 親または猫種または明示キャリアにC制限がない場合、Point/Mink/Sepiaを勝手に出してはならない

### 4.5 W Locus

- W/-: White
- w/w: 有色

不可逆ルール:

- w/w × w/w から W/- は生まれない

### 4.6 S Locus

- S/-: 白斑あり
- s/s: 白斑なし

不可逆ルール:

- s/s × s/s から白斑ありは生まれない

normalでの出力:

- 一般白斑は `-White`
- `Van`, `Mitted`, `Bi-Color` は明示または猫種文脈がない限り出さない

### 4.7 Mc Locus

- Mc/-: Mackerel
- mc/mc: Classic

不可逆ルール:

- mc/mc × mc/mc から Mc/- は生まれない

normalでの出力:

- 親入力に Mackerel / Classic が明示されない場合、通常表示では単に `Tabby` へ簡略化してよい

### 4.8 Ta Locus

- Ta/-: Ticked
- ta/ta: 通常タビー

不可逆ルール:

- ta/ta × ta/ta から Ta/- は生まれない

normalでの出力:

- 猫種が Abyssinian / Somali 等でない限り、Tickedを勝手に出さない

### 4.9 Sp Locus

- Sp/-: Spotted
- sp/sp: 非スポット

不可逆ルール:

- sp/sp × sp/sp から Sp/- は生まれない

### 4.10 I Locus

- I/-: Silver / Smoke
- i/i: 非シルバー

不可逆ルール:

- i/i × i/i から Silver / Smoke は生まれない

### 4.11 Wb Locus

- Wb/-: Wide Band
- wb/wb: 通常幅

不可逆ルール:

- wb/wb × wb/wb から Chinchilla / Shaded / Shell / Golden 系を出してはならない

## 6. 出力名ルール

### 6.1 基本方針

出力名は以下の順に決定する。

1. 内部遺伝子型から標準表現型を算出
2. normal / explicit_carrier / carrier_exploration のモードを確認
3. 性別制約を適用
4. 猫種がある場合は猫種別表示名へ変換
5. 猫種がない場合は一般表示名に正規化
6. 出力禁止語・文脈不一致語を除外
7. 表示確率を集計

### 6.2 CSVに存在する名前の扱い

CSVに存在するカラー名であっても、以下に該当する場合はnormal結果に出してはならない。

- 親の表現型から導出できない
- 明示キャリアがない
- 猫種文脈がない
- 性別と矛盾する
- 白斑レベルが明示されていない
- 通常表示では猫種固有名に該当する

### 6.3 normalで禁止する代表例

- Mink
- Sepia
- Point、明示C条件なしの場合
- Mitted、猫種文脈なしの場合
- Bi-Color、猫種文脈なしの場合
- Van、親または白斑レベル明示なしの場合
- Champagne
- Platinum
- Sable
- Choco省略表記
- male Tortie
- male Tortoiseshell
- male Calico
- male Blue Cream
- male Lilac Cream

### 6.4 ワイドバンド (tipping) 系の命名

ワイドバンド (`Wb/-`) は **非オレンジ・アグーチ背景でのみ tipping (Shell / Shaded / Chinchilla / Golden) として発現**する。`normal_mode` で Wb キャリアを自動展開はしない (§4.11 不可逆ルール) が、親がワイドバンドで子が実際に `Wb/-` になる場合は、その子を**未分類にせず命名する**。

- **非オレンジ・アグーチ・`Wb/-`**: tipping 色として命名する。
  - シルバー (`I/-`) → `Silver`、非シルバー (`i/i`) → `Golden`。希釈 (`d/d`) は `Blue` を冠する。
  - tipping の**濃淡** (Shell < Shaded < Chinchilla) は多遺伝子 (rufousing/wideband polygene) で 1 座位に還元できないため、**親カラー名から推論**する (タビー柄 Mackerel/Classic を親名から決めるのと同方針)。親が `Chinchilla` / `Shaded` / `Shell` を含めばそれを冠し、無ければ濃淡なし (`Golden` / `Silver`)。
  - 例: `Chinchilla Golden × Black` → 非オレンジの子に `Chinchilla Golden` / `Blue Chinchilla Golden` が出る。
- **オレンジ / トーティ / ソリッドの `Wb/-`**: tipping は固有名を持たないため通常命名へ流す (ソリッドでは tipping 非発現、赤系は `Cameo` / `Cream` 等で命名)。
  - 例: `Chinchilla Golden × Cream Tabby-White` は赤ダムのため子が全て赤/トーティになり、純 `Golden` は遺伝的に出ない (`Cameo` / `Patched Tabby` 等で命名され、未分類にはならない)。

> tipping は本来ポリジーンで連続的なため、ここでの命名は「Wb/- の子を消さずに代表名へ寄せる」実務的規約である。母数となる遺伝モデルの厳密化は将来拡張。

## 7. 診断情報

分類不能な子猫遺伝子型を無言で破棄してはならない。

内部診断として以下を保持する。

```json
{
  "diagnostics": {
    "mode": "normal",
    "matched_probability": 100.0,
    "unmatched_probability": 0.0,
    "unmatched_genotype_count": 0,
    "unmatched_samples": []
  }
}
```

### 7.1 再正規化ルール

`unmatched_probability > 0` の状態で、残った結果だけを100%に再正規化してはならない。

通常結果として100%表を返してよいのは、以下の場合のみ。

- `unmatched_probability == 0`
- または未分類を明示的に `Unclassified` として返す

## 8. 確率表示ルール

### 8.1 0.0%禁止

丸め後に `0.0%` となる項目は通常UIに表示しない。

API内部で保持する場合は、以下のいずれかとする。

- `raw_probability`
- `probability_pct`
- `display_probability`

通常表示では `min_display_probability_pct` 未満を除外する。

### 8.2 合計確率

表示結果の合計は100%になること。

ただし、未分類結果を捨てて再正規化した100%は不合格とする。

## 9. API仕様

> 実装状況 (Phase 1): `POST /api/v1/calculate`。`mode` は `normal` / `explicit_carrier` を
> サポート。`explicit_carrier` では `sire_carriers` / `dam_carriers` で開ける座位を指定する。
> 後方互換のためリクエストの `mode`/`*_carriers` は任意 (既定 normal)、レスポンスは従来の
> `status`/`results` に加え `mode`/`diagnostics` を追加 (additive)。

### 9.3 目標カラーから探す API (逆引きMVP)

`POST /api/v1/reverse-lookup` は、登録猫または自舎カラー構成と目標カラーから、目標カラーが
成立し得る交配候補を返す。既存 `POST /api/v1/calculate` の互換性を壊さないため、新規
エンドポイントとして追加する。

リクエストは `target_color` を必須とし、`target_sex` (`male` / `female`) は任意とする。
`target_sex` が指定された場合、確率は該当性別の子猫結果だけを目標条件として集計する。
登録猫の `color` は通常シミュレーターと同じ `cat_color_master.csv` の alias / canonical 解決を通し、
親の性別と `SexRestriction` が矛盾する場合は候補計算へ進まず 422 とする。保存済みデータやAPI直送でも、
父猫にメス限定カラーを指定するなどの不整合を黙って候補なしへ落とさない。

逆引きでは、通常シミュレーターの `normal` 結果とは別に、以下を分けて表示する。

- `confirmed_probability_pct`: 登録済み毛色・確認済み因子だけで評価できる確定確率
- `conditional_max_probability_pct`: 未確認因子が成立すると仮定した条件付き最大確率
- `confirmation_needed`: 確認が必要な条件
- `recommended_tests`: 推奨検査
- `locus_evidence`: 座位別根拠
- `other_possible_colors`: 目標カラー以外に生まれる可能性のあるカラー

逆引き結果は、UIで以下のカテゴリに分けて表示する。

1. 確定で期待できる
2. 条件付きで期待できる
3. 現在の情報では判定が難しい
4. 現在の登録情報では確認できない

候補が見つからない場合も、単に候補なしとはせず、目標カラーに必要な主な条件、現在確認できない
条件、確認するとよい項目を返す。ゴールデン / ワイドバンド / CORIN 系は品種・系統差が大きいため、
現在の対応範囲では既存ルールに基づく参考情報として扱い、断定しすぎない。

### 9.4 リター実績から推定 API (MVP)

`POST /api/v1/litter-inference` は、父猫・母猫の表現型と実際に生まれた子猫群のカラー・性別から、
観察実績を説明できる父母遺伝子型候補を絞り込む。既存 `POST /api/v1/calculate` と
`POST /api/v1/reverse-lookup` の互換性を壊さないため、新規エンドポイントとして追加する。

親猫と子猫の `color` は通常シミュレーターと同じ `cat_color_master.csv` の alias / canonical 解決を
通す。親候補生成・Punnett 分離・子猫遺伝子型の構築は `CoatColorCalculator` の共通処理を使い、
色名・遺伝子座の読み込み先を別系統にしない。

レスポンスは、観察された全子猫を説明できる親候補ペアを基準に、座位別結果を以下へ分類する。

- `confirmed`: 全候補で共通するため確定できる因子
- `conditional`: 遺伝子型は割れるが、観察実績を説明するうえで共通する条件
- `inferred`: 観察実績から強く支持される推定
- `unconfirmed`: 候補が割れて追加確認が必要な因子
- `contradictions`: どの候補でも説明できない矛盾
- `warnings`: Red / Cream 系のA座位判定、Calico / Tortie 系の白斑判定などの注意

MVPでは、タビー細分 (Mc / Ta / Sp) は既存命名ロジック同様に親名・表示後処理の影響が大きいため、
観察実績の主照合から外す。Calico 系で白斑呼称が混在する場合は、S座位を断定せず警告と推奨確認を返す。

## 10. 130 × 204 回帰テスト

### 10.1 入力

- 父: Silver Tabby、コード130
- 母: Blue Pt Tabby-White、コード204
- breed: Any
- mode: normal

### 10.2 禁止出力

以下を含む結果が出た場合はテスト失敗。

- Mink
- Sepia
- Point、明示C条件なしの場合
- Mitted
- Bi-Color
- Bi-C
- Van
- Champagne
- Platinum
- Sable
- Chocolate、明示B条件なしの場合
- Cinnamon、明示B条件なしの場合
- Lilac、明示B/D条件なしの場合
- Fawn、明示B/D条件なしの場合
- Choco
- Lynx Point
- Blue Cream
- Lilac Cream
- **Smoke 系** (a/a 前提。A を normal_mode で展開しないため出ない)
- **Tortie / Tortoiseshell / Calico / Blue Cream 系** (a/a 前提のソリッドトーティ。希釈トーティの正規名は Cream。O/o のメスは Patched Tabby として出る)
- **Solid 単独色** (Black / Blue / Red / Cream / Silver / Cameo 等)。全出力がタビー / パッチドタビー系 (名前に "Tabby" を含む) であること

### 10.3 オス禁止

オス結果に以下を含めてはならない。

- Tortie
- Tortoiseshell
- Calico
- Blue Cream
- Lilac Cream

### 10.4 必須検証

- `probability_pct > 0`
- `unmatched_probability == 0`
- 表示結果合計が100%
- NGカラーが出ない
- オスにトーティ系が出ない

## 11. 完了定義

シミュレーター正本におけるフェーズ1完了は、以下をすべて満たす状態とする。

1. normal / explicit_carrier / carrier_exploration が分離されている
2. normal結果に未明示キャリア由来カラーが混入しない
3. 性別矛盾がない
4. 猫種固有呼称が文脈なしに出ない
5. 未分類遺伝子型が診断できる
6. 0.0%表示がない
7. 不可逆ルールの単体テストがある
8. 130×204回帰テストが通る
9. 既存API互換性を壊す変更がある場合はバージョン付きで管理されている
