# 猫の遺伝学シミュレーター データ正本 V9
作成日: 2026-06-24  
位置づけ: **CSVマスタ・表示名・データ検証ルールの唯一正本**

---

## 0. 本書の目的

本書は、猫の遺伝学シミュレーターで使用するデータマスタの唯一正本である。

対象は以下のマスタである。

1. `cat_color_master.csv` — **色柄概念の唯一正本** (名称・別名・分類・団体差・猫種固有呼称)。仕様は `cat_color_master_schema.md`、差分レビューは `cat_color_master_review.md`。alias 解決は専用カラム `CanonicalColorId` を機械可読の唯一根拠とする。
2. `cat_color_genetic_map.csv` — 遺伝子座定義 (当面エンジンが参照)。`cat_color_master.csv` と段階的に整合させる。
3. `cat_breed_genetic_map.csv`
4. `cat_color_display_alias_map.csv`

V9では、遺伝定義と表示名定義を分離する。  
理由は、同一または近似する遺伝条件が、猫種や登録団体の文脈によって異なる表示名を持つためである。

## 1. データ設計の原則

### 1.1 コードに固定値を書かない

カラー定義、猫種制約、表示名変換、エイリアスは原則としてCSVに保持する。

ただし、以下のような純粋な遺伝評価順序はコード側のロジックとして保持してよい。

- Wが最上位で全色を隠蔽する
- Oは伴性遺伝である
- D/dはd/dで希釈する
- CはPoint / Sepia / Minkを制御する

### 1.2 遺伝定義と表示名を混ぜない

`cat_color_genetic_map.csv` は遺伝条件を定義する。  
`cat_color_display_alias_map.csv` は表示名、猫種別呼称、別名、表示可否を定義する。

### 1.3 空欄の意味

空欄は「未定義」または「制限なし」を意味する。  
空欄をデフォルト遺伝子型で埋めて、通常計算に使用してはならない。

### 1.4 優性表現型のヘテロ未確定ルール (CSV値の解釈)

CSV の遺伝子型値は「その毛色を成立させる代表遺伝子型」であり、親候補の生成時には
シミュレーター正本 §2.4「優性表現型のヘテロ未確定ルール」に従って解釈する。

> 優性形質が表現されている遺伝子座は、表現型のみから優性ホモ接合であるとは断定しない。
> たとえば、タビーは A/A ではなく A/-、濃色は D/D ではなく D/-、シルバーは I/I ではなく I/- として扱う。
>
> 通常モードでは、これらの「表現型を成立させる優性アレルのヘテロ可能性」は計算対象に含める。
> 一方で、チョコレートキャリア B/b、シナモンキャリア B/bl、ポイントキャリア C/cs、セピアキャリア C/cb など、
> 表現型から直接必要とされない潜在キャリアは、明示キャリア情報または血統・産子履歴がない限り通常結果には含めない。

- 親候補生成では、CSV が優性ホモ (D/D, A/A, I/I, Mc/Mc, Ta/Ta) で記載している座を X/- = {X/X, X/x} に展開する (カテゴリA)。
- 劣性ホモ (d/d, a/a, cs/cs, cb/cb, i/i, s/s 等) は固定する (カテゴリB)。
- B/b, B/bl, C/cs, C/cb は CSV に明示が無い限り展開しない (カテゴリC)。
- S (白斑) は不完全優性のため CSV 値 (S/s=-White, S/S=Van, s/s=無印) のまま固定し、X/- 展開しない。

> 逆引き表示名マスタ (cat_color_display_alias_map.csv) は CSV の代表遺伝子型から構築するため、
> 上記の親候補展開とは独立である。S/S(Van) を S/s へ展開して表示名マスタに混ぜてはならない
> (S/s の子が Van 名へ誤マッチするため)。

## 2. cat_color_genetic_map.csv

### 2.1 目的

毛色・柄の内部遺伝条件を定義する。

このCSVは、最終表示名を決める唯一の根拠ではない。  
最終表示名は、表示名マスタと猫種文脈を通して決定する。

### 2.2 必須カラム

| カラム | 型 | 必須 | 説明 |
|---|---|---|---|
| ColorCode | int | 必須 | 色柄コード |
| CoatColor | string | 必須 | 現行の表示名または基準名 |
| CanonicalPhenotype | string | 必須 | 内部標準表現型名 |
| SexRestriction | enum | 任意 | male_only / female_only / both |
| B_Locus | string | 任意 | B/B, B/b, b/b, bl/bl 等 |
| D_Locus | string | 任意 | D/D, D/d, d/d |
| A_Locus | string | 任意 | A/A, A/a, a/a |
| O_Locus | string | 任意 | O/O, O/o, o/o。オスはコード側でO/Y等へ変換 |
| C_Locus | string | 任意 | C/C, C/cs, cs/cs, cb/cb, cb/cs 等 |
| W_Locus | string | 任意 | W/W, W/w, w/w |
| S_Locus | string | 任意 | S/S, S/s, s/s |
| Mc_Locus | string | 任意 | Mc/Mc, Mc/mc, mc/mc |
| Ta_Locus | string | 任意 | Ta/Ta, Ta/ta, ta/ta |
| Sp_Locus | string | 任意 | Sp/Sp, Sp/sp, sp/sp |
| I_Locus | string | 任意 | I/I, I/i, i/i |
| Wb_Locus | string | 任意 | Wb/Wb, Wb/wb, wb/wb |
| IsGeneralDisplayAllowed | bool | 必須 | Any/一般表示で出してよいか |
| IsBreedSpecificOnly | bool | 必須 | 猫種文脈専用名か |
| Notes | string | 任意 | 補足 |

### 2.3 複数許容値の書式

複数の遺伝条件を許可する場合は `|` 区切りとする。

例:

```csv
Ta/Ta|Ta/ta
```

カンマはCSV構造と衝突するため禁止する。

### 2.4 SexRestriction

| 値 | 意味 |
|---|---|
| both | オス・メス両方 |
| male_only | オスのみ |
| female_only | メスのみ |

以下は原則として `female_only` とする。

- Tortoiseshell
- Tortie
- Calico
- Blue Cream
- Lilac Cream
- Tortie Point
- Blue Cream Point
- Lilac Cream Point

通常XYオス例外はフェーズ1対象外。

## 3. cat_breed_genetic_map.csv

### 3.1 目的

猫種ごとに100%固定される遺伝子条件のみを定義する。

このCSVは、猫種の「許容カラー一覧」ではない。  
猫種で本当に固定される遺伝子座だけを書く。

### 3.2 必須カラム

| カラム | 型 | 必須 | 説明 |
|---|---|---|---|
| BreedCode | int | 必須 | 猫種コード |
| Breed | string | 必須 | 猫種名 |
| BreedGroup | string | 任意 | SH/LH等の区分 |
| B_Locus | string | 任意 | 固定される場合のみ |
| D_Locus | string | 任意 | 固定される場合のみ |
| A_Locus | string | 任意 | 固定される場合のみ |
| C_Locus | string | 任意 | 固定される場合のみ |
| W_Locus | string | 任意 | 固定される場合のみ |
| S_Locus | string | 任意 | 固定される場合のみ |
| Mc_Locus | string | 任意 | 固定される場合のみ |
| Ta_Locus | string | 任意 | 固定される場合のみ |
| Sp_Locus | string | 任意 | 固定される場合のみ |
| I_Locus | string | 任意 | 固定される場合のみ |
| Wb_Locus | string | 任意 | 固定される場合のみ |
| DisplayColorProfile | string | 任意 | 表示名変換プロファイル |
| Notes | string | 任意 | 補足 |

### 3.3 入れてよい値

猫種制約には「本当にその猫種で固定される遺伝子」だけを入れる。

例:

| 猫種 | 入れてよい例 | 理由 |
|---|---|---|
| Siamese | C_Locus = cs/cs | ポイント固定 |
| Burmese | C_Locus = cb/cb | セピア固定 |
| Tonkinese | C_Locus = cb/cs | ミンク固定 |
| Russian Blue | D_Locus = d/d, B_Locus = B/B, A_Locus = a/a | ブルー単色固定 |
| Chartreux | D_Locus = d/d | ブルー系固定 |
| Abyssinian | Ta_Locus = Ta/Ta\|Ta/ta | ティックド前提 |

### 3.4 入れてはいけない値

多くの猫種で許容される単なる「よくある毛色」を固定条件として入れてはならない。

例:

- Munchkin に A/A を固定しない
- Munchkin に s/s を固定しない
- Maine Coon にD/Dを固定しない
- Persian に特定カラーを固定しない
- Scottish Fold に耳形質を入れない
- Sphynx に毛質遺伝子をフェーズ1カラー計算へ入れない

### 3.5 O_Locusの扱い

猫種制約では、原則としてO_Locusを固定しない。

理由:

- Oは伴性遺伝
- オスはO/Yまたはo/Y
- メスはO/O, O/o, o/o
- 猫種制約で o/o のように固定すると、オス判定で事故る

O系の制限が必要な場合は、猫種制約ではなくカラー許容や表示名変換で扱う。

## 4. cat_color_display_alias_map.csv

### 4.1 目的

遺伝的には同じ、または近い内部表現型を、猫種や表示文脈に応じて正しい表示名へ変換する。

### 4.2 必須カラム

| カラム | 型 | 必須 | 説明 |
|---|---|---|---|
| AliasId | int | 必須 | 行ID |
| CanonicalPhenotype | string | 必須 | 内部標準表現型 |
| GeneralDisplayName | string | 必須 | 一般表示名 |
| Breed | string | 任意 | 猫種名。Anyの場合は空欄またはAny |
| BreedSpecificDisplayName | string | 任意 | 猫種別表示名 |
| Registry | string | 任意 | TICA, CFA等 |
| OfficialStatus | enum | 必須 | official / non_official / deprecated / internal_only |
| DisplayContext | enum | 必須 | general / breed_specific / carrier_exploration / hidden |
| Priority | int | 必須 | 同候補競合時の優先度 |
| Notes | string | 任意 | 補足 |

### 4.3 DisplayContext

| 値 | 意味 |
|---|---|
| general | Anyまたは一般表示で使用可 |
| breed_specific | 指定猫種時のみ使用 |
| carrier_exploration | 全キャリア探索時のみ使用 |
| hidden | 内部処理のみ。通常表示不可 |

### 4.4 変換例

| CanonicalPhenotype | Breed | 表示名 |
|---|---|---|
| Brown Ticked Tabby | Abyssinian | Ruddy |
| Blue Ticked Tabby | Abyssinian | Blue |
| Cinnamon Ticked Tabby | Abyssinian | Cinnamon |
| Fawn Ticked Tabby | Abyssinian | Fawn |
| Seal Point | Siamese | Seal Point |
| Seal Point Mitted | Ragdoll | Seal Point Mitted |
| Brown Tabby | Any | Brown Tabby |
| Bronze | Any | Brown Tabby、Bronzeは非優先 |

## 5. 白斑表示名ルール

### 5.1 一般表示

Anyまたは猫種未指定の場合、白斑ありは原則として `-White` に正規化する。

### 5.2 Van

`Van` は以下の場合のみ出力可。

- 親入力にVanが明示されている
- ユーザーが白斑レベルを明示している
- 猫種別表示名マスタでVanが明示的に許可されている

### 5.3 Mitted

`Mitted` は猫種固有表示として扱う。

Anyや一般表示では出さない。

### 5.4 Bi-Color

`Bi-Color` は猫種固有または登録団体文脈の表示として扱う。

Anyでは原則 `-White` に正規化する。

---

## 6. Point / Mink / Sepia 表示ルール

### 6.1 normalで出してよい条件

以下のいずれかを満たす場合のみ出力可。

- 親入力がPoint / Mink / Sepia / Lynx Point等を明示している
- 明示キャリアモードでC_Locusが指定されている
- 猫種制約でC_Locusが固定されている

### 6.2 normalで出してはいけない条件

親の見た目にC制限がなく、明示キャリアもなく、猫種制約もない場合は出力禁止。

禁止例:

- Seal Point
- Blue Point
- Chocolate Point
- Lilac Point
- Mink
- Sepia
- Lynx Point
- Champagne Mink
- Platinum
- Sable

## 7. データ検証ルール

### 7.1 CSVロード時チェック

すべてのCSVはロード時に以下を検証する。

- 必須カラムが存在する
- 遺伝子座カラムの値が許容書式に一致する
- 複数許容値は `|` 区切り
- カンマ区切り複数値を禁止
- SexRestrictionが定義値内である
- DisplayContextが定義値内である
- OfficialStatusが定義値内である

### 7.2 禁止データ

以下はエラーまたは警告にする。

- male_only と Calico の矛盾
- female_only なのに male候補へ展開
- Any表示でMittedを許可
- Any表示でBi-Colorを許可
- Any表示でSable/Champagne/Platinumを許可
- 通常表示でChoco省略名を使う
- O_Locusを猫種制約に無条件で固定する

## 8. データ正本の完了条件

データ正本は以下を満たすとフェーズ1完了とする。

1. 遺伝定義CSVと表示名CSVが分離されている
2. 猫種CSVは固定遺伝子のみを保持している
3. Any表示で猫種固有名が出ない
4. normal表示で未明示C系が出ない
5. オスにトーティ系表示が割り当てられない
6. 0.0%になる結果が通常表示に出ない
7. 130×204の禁止語テストを通過する
8. CSVロード時バリデーションがある
