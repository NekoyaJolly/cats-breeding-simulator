## 2. 展開カテゴリ（normalモードの遺伝子型展開規則）

出典: `01_シミュレーター正本_V9.md` §2.4。

- **A（優性ヘテロ未確定・展開する）**: 優性形質が見えてもホモ/ヘテロ不明。`X/- = {X/X, X/x}` を両方計算（中立仮定 50:50）。対象: **D・I・Mc・Ta**。
- **A'（タビーA座・展開しない）**: A/- は normal では **A/A 相当に固定**し A/a を展開しない（a/a×a/a 由来の Solid/Tortie 等の誤出力を防ぐ）。
- **B（劣性固定）**: 表現型から確定する劣性ホモは固定。d/d・a/a・cs/cs・cb/cb・cb/cs・i/i・s/s。
- **C（潜在キャリア・展開しない）**: 表現型から直接要求されない潜在キャリアは normal で展開しない。B/b・B/bl・C/cs・C/cb。明示キャリア（explicit_carrier）または血統・産子履歴がある場合のみ扱う。

## 4. タビー3座位（Mc / Ta / Sp）の関係

タビーの見た目は**1本の対立遺伝子ラダーではなく、3座位のエピスタシス**で決まる。

```
Ta座位（ティックド）: Ta/- がエピスタシスで最上位 → 下位の縞柄を覆い地色ティッキングにする
   └ ta/ta のとき下位が出る
       Mc座位（地の縞型）: Mc/-（マッカレル） ＞ mc/mc（クラシック）
           └ Sp座位（修飾）: Sp/- がマッカレル地（Mc/-）を斑点に割る = スポテッド
```

見た目の優先順: **Ticked（Ta/-） ＞ [ Spotted（Mc/- 地 ＋ Sp/-） ｜ Mackerel（Mc/-） ｜ Classic（mc/mc） ]**

- Sp は **マッカレル地（Mc/-）を前提とする修飾**であり、クラシック（mc/mc）と同列の対立遺伝子ではない。
- さらに上位の a/a（ソリッド）・W/-（白）でタビー柄自体がマスクされる。

## 5. 座位別定義

各座位: 対立遺伝子 / 不可逆ルール / 発現・推論条件 / 猫種固定 / 実装状況。計算の詳細は `01_シミュレーター正本_V9.md` §4 を正とする。

### 5.1 A — アグーチ（タビー）
- A/-: タビー許可、a/a: ソリッド。
- 不可逆: 非オレンジの a/a × a/a から A/- は生まれない。オレンジ系列はAに関係なくタビー様表現が顕在化。
- normal: A/- は A/A 固定（カテゴリA'、A/a 非展開）。
- 実装済。

### 5.2 B — ブラック系 ★express-then-infer
- B ＞ b（チョコレート） ＞ bl（シナモン）。優先順: `_ALLELE_PRIORITY` B={B:0,b:1,bl:2}。
- 表現: 色名で表現（Black=B/-, Chocolate=b/b, Cinnamon=bl/bl, Lilac=b/b+d/d, Fawn=bl/bl+d/d）。色マップに `B_Locus` 列なし。
- **発現・推論原則（express-then-infer）**: 劣性（b/b・bl/bl）が**表現された個体からのみ**根拠を取る。黒に勝手に `B/-`（隠れキャリア）を被せない。3コンテキストの扱いは §6 を参照。
- 猫種固定: Bombay/Chartreux/Korat 等 = `B/B`（`cat_breed_genetic_map.csv`）。
- 実装済（色マップは列なし＝色名駆動）。

### 5.3 C — 発色
- C/-: フルカラー、cs/cs: Point、cb/cb: Sepia、cb/cs: Mink。優先順: C={C:0,cb:1,cs:2}。
- 不可逆: cs/cs × cs/cs / cb/cb × cb/cb から C/- は生まれない。
- normal: 親・猫種・明示キャリアにC制限がなければ Point/Mink/Sepia を出さない。C/cs・C/cb はカテゴリC（潜在キャリア非展開）。
- 猫種固定: Siamese / Balinese / Ragdoll / Snowshoe 等 = `cs/cs`、Burmese / European Burmese / Singapura 等 = `cb/cb`。Tonkinese は猫種全体では固定せず、色クラスから Point=`cs/cs`、Mink=`cb/cs`、Solid/Sepia=`cb/cb` として扱う。
- 実装済。

### 5.4 D — 希釈
- D/-: 濃色、d/d: 希釈。優先順: D={D:0,d:1}。
- 不可逆: d/d × d/d から D/- は生まれない。
- normal: D/- はカテゴリA（D/D:D/d = 50:50 で展開）。
- 実装済。

### 5.5 I — シルバー（インヒビター）
- I/-: Silver / Smoke、i/i: 非シルバー。優先順: I={I:0,i:1}。
- 不可逆: i/i × i/i から Silver/Smoke は生まれない。
- normal: I/- はカテゴリA（展開）。
- 実装済。

### 5.6 O — オレンジ（X連鎖）
- オス: O/Y または o/Y。メス: O/O・O/o・o/o。♀ヘテロ O/o は共優性的にトーティ。優先順: O={O:0,o:1,Y:2}。
- 不可逆: o/Y × o/o から Red/Cream/Tortie/Calico は生まれない。通常XYオスに Tortie/Tortoiseshell/Calico/Blue Cream/Lilac Cream を出してはならない。
- 実装済。

### 5.7 S — 白斑（特例）
- S/S: Van（高白斑）、S/s: -White（バイカラー）、s/s: 白斑なし。**不完全優性**で接合性が表現型から確定。
- 特例: X/- 展開しない。入力名で固定（"White"=S/s, "Van"=S/S, 無印=s/s）。逆引きの Van 誤マッチ回避。
- 不可逆: s/s × s/s から白斑ありは生まれない。
- 出力: 一般白斑は `-White`。Van/Mitted/Bi-Color は明示または猫種文脈がない限り出さない。
- 実装済。

### 5.8 W — 優性白
- W/-: White（全色柄をエピスタシスでマスク）、w/w: 有色。
- 不可逆: w/w × w/w から W/- は生まれない。
- 実装済。

### 5.9 Mc — 縞型
- Mc/-: マッカレル、mc/mc: クラシック。
- 不可逆: mc/mc × mc/mc から Mc/- は生まれない。
- normal: Mc/- はカテゴリA（展開）。親に Mackerel/Classic の明示がなければ通常表示は単に `Tabby` へ簡略化してよい。
- 実装済。

### 5.10 Ta — ティックド
- Ta/-: ティックド、ta/ta: 通常タビー。エピスタシスで Mc/Sp 柄を覆い地色ティッキングにする。
- 不可逆: ta/ta × ta/ta から Ta/- は生まれない。
- normal: Ta/- はカテゴリA（展開）。猫種が Abyssinian/Somali 等でない限り Ticked を勝手に出さない。
- 猫種固定: Abyssinian/Somali = `Ta/Ta`。
- 実装済。

### 5.11 Sp — スポテッド ★未実装（Phase B）
- Sp/-: スポテッド（マッカレル地 Mc/- を斑点に割る修飾）、sp/sp: 非スポット。
- 不可逆: sp/sp × sp/sp から Sp/- は生まれない。
- **発現・推論条件**: タビー同様 express-then-infer。**表現型に出ている時 ∨ 猫種固定**でのみ確定。Ta（ティックド）と同型の「猫種依存座位」。
  - **猫種固定（Sp/Sp）**: Egyptian Mau・Ocicat（常時スポット）。← 確定（2026-06-30）
  - **固定しない**: Bengal（スポット優勢だがマーブル=mc/mc 個体あり。「重み付け」は §7 注記の将来機能）、Bengal(A.O.C)、Toyger（標準はマッカレル）、その他。これらのスポット個体は色行（Spotted Tabby）で表現する。
- エピスタシス: §4 のとおり Ta/- に覆われ、Mc/- 地を前提とし、a/a・W/- でマスクされる。
- **実装状況（実装済 2026-06-30）**: 両CSVに `Sp_Locus` 列を追加（色マップ: スポット色15行=`Sp/Sp`、他=`sp/sp` ／ 猫種マップ: Egyptian Mau・Ocicat=`Sp/Sp`）。エンジンは既に Sp を `AUTOSOMAL_LOCI` に持ち、ローダーが `_Locus` 列を総称読込するため**コード改修ゼロ**で機能。Mau/Ocicat は非スポット色を認定カラー外として拒否し、スポット色は出力に保持される。Bengal/Toyger は固定せず色行で表現。柄命名は従来どおり親カラー名＋猫種制約で駆動（遺伝子型レベルの Sp ゼロ接合性推論は将来のキャリア推論エンジンと同領域）。

### 5.12 Wb — ワイドバンド
- Wb/-: ワイドバンド、wb/wb: 通常。優先順: Wb={Wb:0,wb:1}。
- 発現: 非オレンジ・アグーチ背景でのみ tipping（Shell/Shaded/Chinchilla/Golden）として発現。シルバー（I/-）と組めばチンチラ/シェーデッド、シルバー無しならゴールデン。
- 不可逆: wb/wb × wb/wb から Chinchilla/Shaded/Golden 系は生まれない。
- normal: Wb キャリアを自動展開しない。親がワイドバンドで子が実際に Wb/- になる場合は未分類にせず命名する。
- 実装済。

## 6. B / Sp の express-then-infer 原則（3コンテキスト）

B（チョコ/シナモン）と Sp（スポット）は、いずれも**表現された劣性ホモ／猫種文脈からのみ根拠を取る**。推測で隠れキャリアを被せない。

| コンテキスト | B（チョコ/シナモン） | Sp（スポット） |
|--------------|----------------------|----------------|
| 通常（親→子） | 親の毛色自体が劣性表現（b/b・bl/bl）の時のみ b を計算に入れる。黒に `B/-` を勝手に被せない | 親の柄がスポット表現、または猫種がスポット定義品種の時のみ Sp を入れる |
| リター推定（子→親） | チョコ/シナモンの子が出たら両親は必ず b を供出 → **両親キャリア確定**（黒×黒→チョコなら両親 `B/b` 確定） | スポットの子が出たら、表現された親または親のいずれかが Sp 供出元 → 表現に基づき確定 |
| 逆算（目標→親ペア） | 黒の b 保有は**未知=conditional**。チョコ/ライラック等の表現個体が居て初めて確定経路（ライラック×黒は「黒が b 持ちなら」conditional） | スポット定義品種または表現個体が居て初めて確定経路。非表現の黒/タビーに Sp を仮定しない |

この原則は既存の `reverse_lookup`（confirmed / conditional 分離）・`litter_inference`（confirmed / inferred）の枠にそのまま乗る。SaaS化の `CarrierEvidence` 台帳（推定証跡）もこの原則で記録する。
